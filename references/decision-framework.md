# 决策辅助框架

本文档为 G2 在决策辅助模式下提供具体的思维工具和提问锚点。决策辅助是 G3 主导的模式——认知偏差在犹豫时刻最活跃，这份文档确保每次都能系统性地暴露偏差。

## 一、澄清决策

用户说"我在纠结要不要买/卖"时，先帮他把模糊的焦虑具体化。

**关键提问**：
- 你到底在纠结什么？是方向（买还是不买），还是时机（现在还是等等），还是仓位（买多少）？
- 如果必须在今天做决定，你会怎么选？为什么？
- 你期望的结果是什么？你害怕的结果是什么？

**目标**：把"我很纠结"变成"我在A和B之间犹豫，因为X"。

## 二、事前验尸（Pre-mortem）

这是最强的去偏差工具。不是问"会不会失败"，而是**假设已经失败了，倒推原因**。

**必问的两个问题**：

> "假设一年后这笔投资亏了50%，最可能的原因是什么？"

让用户列出3-5个可能的原因。这会强制激活他被确认偏差压制的风险意识。

> "假设一年后你非常后悔没买，最可能的原因是什么？"

这是反向的事前验尸，防止损失厌恶导致的过度保守。

**深入追问**：
- 这些原因中，哪个你之前没认真想过？
- 如果亏损的原因是你现在能预见的，为什么你还愿意承担这个风险？
- 亏50%后你会怎么做？（如果答案是"加仓"，追问加仓的资金从哪来、加仓的逻辑是什么）

## 三、反转测试

用来打破禀赋效应和沉没成本。

**必问的两个问题**：

> "如果你现在没有持有这只股票，以今天的价格你会买入吗？"

如果答案是"不会"——那你为什么还在持有？唯一的区别是你已经买了（禀赋效应）。

> "如果你持有的是等值现金而不是这只股票，你会换成股票吗？"

这是同一个问题的另一种问法，但会让用户从不同角度思考。

**变体**（根据情境选用）：
- 卖出纠结："如果今天是第一天看到这只股票，你会在这个价格买吗？"
- 加仓纠结："假设你的仓位是零，你会一次性买到当前仓位的大小吗？"
- 持有纠结："如果这不是你买的，是朋友让你帮忙看的，你会建议继续持有吗？"

## 四、技术面时机判断

决策辅助中的技术面不是做完整的技术分析，而是回答三个具体问题：

1. **当前技术面支持这个方向吗？**
   - 想买但趋势向下 → 左侧交易，风险更大，仓位要小
   - 想卖但趋势向上 → 可能是恐惧驱动的卖出，需要检查损失厌恶

2. **有没有更好的价位可以等？**
   - 支撑位/压力位在哪
   - 是否有可预期的形态完成点（如W底颈线突破）

3. **止损应该设在哪里？**
   - 必须在买入前确定
   - 止损位决定了你最大可能亏损，进而决定仓位大小

## 五、仓位校准

仓位是决策中最容易被忽视但影响最大的变量。

**核心规则**（δ 硬约束）：
- 单只个股≤3成
- 同板块≤5成
- 任何单笔亏损≥8%必须止损

**校准提问**：
- 这个决策与你的整体投资策略一致吗？（还是一时冲动？）
- 你能承受的最大损失是多少？不只是金额，包括心理压力。
- 如果这笔投资亏了，会不会影响你的生活质量？会 → 仓位太大了。
- 你有没有"如果跌到X就加仓"的计划？如果有，加仓的钱从哪来？

## 六、全面偏差扫描

决策辅助的最后一步，对整个决策过程做偏差体检。

### 决策辅助中最高频的偏差

| 偏差 | 在决策时刻的典型表现 | 对策性提问 |
|------|---------------------|-----------|
| 锚定效应 | "从高点跌了这么多，很便宜了" | 忘掉买入价，现在的基本面值这个价吗？ |
| 确认偏差 | 只找支持自己决定的信息 | 你有没有认真找过反面证据？ |
| 损失厌恶 | 亏损仓位死拿不卖 | 如果现在是空仓，你会买吗？（反转测试） |
| 沉没成本 | "已经亏了这么多，现在卖太亏了" | 已经亏的钱回不来了，问题是从现在开始放在哪最好 |
| 从众效应 | "大家都在买/都在卖" | 大家也都买在了高点。人多的方向不一定对 |
| 过度自信 | "我研究得很透了，肯定没问题" | 你的分析在什么条件下会失效？ |
| 近因偏差 | "最近涨得好，应该继续涨" | 最近三个月的趋势能代表未来吗？ |
| 处置效应 | 急着卖赚钱的，死拿亏钱的 | 你的卖出理由是"赚够了"还是"基本面变了"？ |

### 综合建议的输出原则

1. 不给"买"或"不买"的二元结论——给出在什么条件下买、什么条件下不买
2. 明确触发条件："如果X发生，做Y；如果Z发生，做W"
3. 附带仓位建议和止损位
4. 标注建议的失效条件："如果以下情况出现，这个建议不再适用"
